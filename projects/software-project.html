Contrastive Audio Diffusion

Conditional waveform diffusion for audio denoising and instrument-guided generation.

Samuel Li, Zachary Houlton, Ariv Mondal, Daniel Zhu

Code Medium writeup and audio examples

Contrastive Audio Diffusion is a team project built around audio denoising with diffusion models. The Medium writeup is the listening-facing artifact for the project: it explains the diffusion setup and includes the embedded audio snippets for comparing noisy inputs and denoised outputs. This page is the technical companion, focused on the ML design and software architecture behind those experiments.

The codebase extends the original tiny-audio-diffusion starting point into a conditional waveform diffusion system. The main additions are the NSynth data pipeline, instrument-label conditioning, contrastive alignment between label and audio embeddings, reproducible Hydra experiment configs, evaluation scripts, and a Django inference interface.

Data Pipeline

The dataset layer is designed around a stable sample dictionary instead of the raw NSynth folder layout. Each item resolves clean and optional noisy waveform tensors from JSONL metadata, normalizes sample rate and length, and returns the waveform, split, instrument class, class id, and optional conditioning vector through the same interface.

This lets the same model code run on locally prepared subsets, processed dataset exports, and clean-only datasets. When a noisy waveform is missing, the loader falls back to the clean waveform, allowing the training and evaluation pipeline to keep one contract while the dataset source changes.

Model Architecture

The denoiser is a 1D U-Net wrapped by a diffusion sampler. The project compares one-hot conditioning against learnable label embeddings, with optional cross-attention so class information can condition the denoising process without treating the label vector as another audio channel.

The contrastive path aligns audio embeddings with label embeddings during training. In the embedding model, the diffusion loss teaches waveform denoising while the auxiliary contrastive loss pushes matching instrument labels and audio clips closer in latent space and separates mismatched pairs.

Inference And Evaluation

The inference layer reuses the same checkpoint and config artifacts instead of depending on notebook state. The Django demo loads a selected model, accepts either uploaded reference audio or a metadata sample, prepares the target conditioning, runs the sampler, and returns audio outputs with benchmark timing records.

The evaluation tools report waveform error, SNR, SNR improvement, timing, GPU memory, and time-frequency error visualizations. That makes the demo more than playback: it helps identify whether failures are concentrated in transients, sustained harmonics, low-frequency energy, or specific time windows.
